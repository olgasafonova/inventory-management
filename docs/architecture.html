System Architecture

Factory Inventory Management System

A full-stack demo for inventory tracking, order management, demand forecasting, and spending analytics across factory warehouses. Three-tier design: a Vue single-page app, a FastAPI service, and an in-memory JSON data layer.

Frontend  localhost:3000
Backend API  localhost:8001
API Docs  localhost:8001/docs

System Architecture

Three layers with a single direction of dependency. The browser talks only to the API; the API owns all data access and validation. There is no database — data lives in memory for the lifetime of the server process.

Data Flow

A single request, from a filter change in the UI to a validated JSON response rendered as derived state. Filters are the spine of the system — they flow as query params all the way to in-memory list comprehensions.

User changes a filter

A view binds to the useFilters composable (Time Period, Warehouse, Category, Status). Selected values are held in shared refs.

View calls the API client

The view invokes a method on api.js, which serializes active filters into a query string with URLSearchParams — skipping any filter set to all.

Axios issues an HTTP GET

Request goes to http://localhost:8001/api/.... CORS middleware on the server accepts the cross-origin call from port 3000.

FastAPI handler filters in memory

The route reads query params, then runs apply_filters() (warehouse / category / status) and filter_by_month() (direct month or quarter map) over the in-memory lists.

Pydantic validates the response

Results are shaped by response_model (e.g. List[InventoryItem]), guaranteeing the JSON contract before it leaves the server.

Vue stores raw data, derives the rest

The response lands in reactive refs (e.g. allOrders). computed properties derive metrics, chart series, and table rows — re-running only when their dependencies change.
